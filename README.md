# User-Centred Deep Energy Renovation Early-Stage Decision-Making Tool

GitHub Pages–ready static website package.

## Files

- `index.html` — complete interactive tool; logos, styles, data and calculations are embedded.
- `favicon.svg` — browser-tab icon.
- `.nojekyll` — prevents unnecessary Jekyll processing.
- `robots.txt` — permits normal indexing.
- `DEPLOYMENT_CHECKLIST.md` — publishing and verification steps.

## Publish in an existing GitHub repository

1. Open the repository on GitHub.
2. Upload the **contents of this folder** to the repository root. `index.html` must be at the top level.
3. Commit the files to the `main` branch.
4. Open **Settings → Pages**.
5. Under **Build and deployment**, select **Deploy from a branch**.
6. Select branch **main** and folder **/(root)**, then save.
7. Wait for GitHub Pages to publish the site and open the URL shown in the Pages settings.
8. Enable **Enforce HTTPS** when the option appears.

## Updating the site

Replace `index.html` with the revised version and commit the change. GitHub Pages will republish automatically.

## Important

Do not rename `index.html`. Do not upload only the outer ZIP file to the repository; upload the extracted files.
